# 本亦运营后台 V2

这是本亦内部使用的运营后台，用来管理：

- 员工账号登录和角色权限
- 学校作战表
- UGC 活动看板
- 门店日报
- 工作汇总
- 月度汇总
- CSV 导入导出

当前版本本地运行时使用 SQLite 数据库文件，不再使用浏览器 localStorage。

线上部署时支持连接云 PostgreSQL 数据库。这样可以不购买 Render 持久磁盘，员工通过公开网址录入的数据会保存到云数据库里。

如果要部署到线上服务器，让员工通过公开网址访问，请看：

```text
DEPLOY_RENDER.md
```

## 如何安装依赖

本地试用不需要安装复杂依赖，Mac 一般自带 `python3`，直接使用即可。

如果要连接线上 PostgreSQL 数据库，需要先安装依赖：

```bash
pip3 install -r requirements.txt
```

如果不确定有没有 Python，可以打开「终端」输入：

```bash
python3 --version
```

能看到版本号就可以继续。

## 如何启动后端

1. 打开「终端」。
2. 输入下面这行命令进入项目文件夹：

## 如何打开前端

启动后，在浏览器打开：

```text
http://127.0.0.1:8000
```

不要双击 `index.html`，V2 需要通过后端服务访问，登录和数据库才能正常工作。

## 员工如何在同一个 Wi-Fi 里访问

现在已经支持局域网访问。

最简单方式：

1. 双击桌面上的「启动本亦运营后台.command」。
2. 启动窗口会显示一个员工访问地址，例如：

```text
http://192.168.1.23:8000
```

3. 让员工的电脑或手机连接同一个 Wi-Fi。
4. 员工在浏览器输入这个地址。
5. 员工用自己的账号密码登录。

注意：

- 你的 Mac 必须开机。
- 本亦后台必须保持运行。
- 员工必须和你的 Mac 在同一个 Wi-Fi。
- 如果 Mac 弹出网络权限或防火墙提示，请选择「允许」。
- 如果员工打不开，先双击「停止本亦运营后台.command」，再重新双击「启动本亦运营后台.command」。

## 数据库保存在哪里

本地运行时，数据库文件在：

```text
/Users/xuzirui/Documents/Codex/2026-06-06/0-mvp-1-2-3-4/data/benyi_v2.sqlite
```

只要这个文件还在，刷新网页或重启服务后数据都不会丢。

线上部署时，如果设置了 `DATABASE_URL`，数据会保存到云 PostgreSQL 数据库，不再保存到本机 SQLite 文件。

## 如何重置数据

如果想完全重置成初始示例数据：

1. 先在终端按 `Control + C` 停止服务。
2. 删除数据库文件：

```bash
rm "data/benyi_v2.sqlite"
```

3. 重新启动：

```bash
python3 server.py
```

系统会自动重新创建数据库和默认账号。

## 如何导入 CSV

1. 登录后台。
2. 进入「数据导入导出」。
3. 先下载对应模板。
4. 用 Excel 或 Numbers 填好模板。
5. 保存为 CSV。
6. 回到后台选择文件，点击「导入 CSV」。

当前支持：

- 学校 CSV 导入/导出
- UGC 活动 CSV 导入/导出
- 门店日报 CSV 导入/导出
- 工作汇总 CSV 导入/导出

## 角色权限说明

老板 admin：

- 查看全部数据
- 新增、编辑、删除全部数据
- 管理员工账号
- 导入导出全部数据

运营负责人：

- 查看全部学校、UGC、门店日报、工作汇总
- 可以编辑数据
- 不能删除数据
- 不能管理员工账号

学校负责人：

- 只查看和编辑自己负责的数据
- 不能删除数据

文员：

- 可以新增和录入学校、UGC、门店日报
- 不能删除数据
- 不能看财务金额汇总

只读账号：

- 只能查看
- 不能新增、编辑、删除

## 文件说明

- `server.py`：后端服务、登录、权限、数据库、API。
- `index.html`：前端页面入口。
- `styles.css`：后台界面样式。
- `app.js`：前端交互逻辑。
- `data/benyi_v2.sqlite`：本地 SQLite 数据库文件，启动后自动生成。
- `requirements.txt`：线上连接 PostgreSQL 需要的 Python 依赖。
- `render.yaml`：Render 部署配置。
